# Human Oversight Review Checklist

## Purpose

This checklist is intended to support practical human oversight activities for AI-enabled systems, workflows and decision-support processes.

The goal is to encourage structured review, escalation awareness and operational accountability during AI system usage.

---

# Oversight Review Checklist

| Review Item | Status | Notes |
|---|---|---|
| Is a human accountable for reviewing AI outputs? | Yes / No |  |
| Are users aware that AI outputs may contain errors? | Yes / No |  |
| Is there a process for escalation of concerning outputs? | Yes / No |  |
| Are high-risk decisions reviewed by a human? | Yes / No |  |
| Is sensitive data usage monitored? | Yes / No |  |
| Are AI-generated recommendations independently validated where required? | Yes / No |  |
| Is there a documented incident reporting process? | Yes / No |  |
| Are AI limitations communicated to users? | Yes / No |  |
| Is periodic governance review conducted? | Yes / No |  |
| Are unauthorised AI tools monitored or restricted? | Yes / No |  |

---

# Accountability Trigger Examples

Examples of situations that may require human escalation or governance review:

- AI-generated output appears misleading or factually incorrect
- AI recommendation impacts customer eligibility or access
- Sensitive or regulated data is processed unexpectedly
- Human reviewers identify repeated system inconsistencies
- Users rely on AI outputs without appropriate validation
- AI usage falls outside approved governance scope
